# Swanray

Минимальный Electron-клиент для VLESS-серверов с поддержкой **split tunneling** — выбора программ, чей трафик НЕ должен идти через VPN.

В качестве ядра используется [`sing-box`](https://github.com/SagerNet/sing-box). UI на Electron, ядро — отдельный нативный процесс, запускаемый с правами администратора (TUN-режим).

## Возможности

- Подключение к VLESS-серверу из ссылки `vless://...` (поддержка TLS, Reality, uTLS-fingerprint, ws/grpc/h2 транспортов).
- Системный VPN через виртуальный TUN-интерфейс — захватывается весь трафик ОС.
- Split tunneling: укажите `.exe`-файлы (или просто имена процессов) — их трафик пойдёт мимо VPN.
- Сохранение последнего ключа и списка исключений между запусками.
- Просмотр логов sing-box в реальном времени.

## Установка

### 1. Зависимости

```bash
npm install
```

### 2. Загрузка sing-box

Электронное приложение не несёт `sing-box.exe` в комплекте — скачайте его самостоятельно:

1. Перейдите на страницу релизов: https://github.com/SagerNet/sing-box/releases
2. Скачайте архив для Windows, например `sing-box-1.10.0-windows-amd64.zip`.
3. Распакуйте `sing-box.exe` в папку `bin/` этого проекта.

После запуска приложение само подскажет, если файла нет.

### 3. Запуск

```bash
npm start
```

## Как пользоваться

1. Вставьте ключ `vless://...` в верхнее поле.
2. (Опционально) Добавьте программы в раздел «Исключения». Можно выбрать `.exe` файлы через диалог или вписать имя процесса вручную (например `chrome.exe`, `Telegram.exe`).
3. Нажмите **Подключиться**. Windows запросит права администратора (UAC) — они нужны для создания TUN-интерфейса.
4. После подключения весь системный трафик идёт через VPN, кроме программ из списка исключений.
5. Нажмите **Отключиться** для остановки (UAC попросит подтвердить остановку).

## Как работает split tunneling

`sing-box` умеет смотреть, какой процесс инициировал сетевое соединение (по PID → имя exe), и применять разные правила маршрутизации. В сгенерированном конфиге для каждого имени из списка исключений создаётся правило:

```json
{ "process_name": ["chrome.exe", "telegram.exe"], "outbound": "direct" }
```

Соединения от перечисленных процессов отправляются напрямую, всё остальное — через VLESS-прокси.

## Структура проекта

```
.
├── main.js                  # Главный процесс Electron, IPC, управление sing-box
├── preload.js               # Безопасный мост main ↔ renderer
├── renderer/
│   ├── index.html           # UI
│   ├── styles.css
│   └── renderer.js          # Логика интерфейса
├── src/
│   ├── vless-parser.js      # Разбор vless:// URL
│   ├── singbox-config.js    # Генерация config.json для sing-box
│   └── singbox-manager.js   # Запуск/остановка sing-box, чтение логов
├── bin/
│   └── sing-box.exe         # ← Скачайте сами
└── package.json
```

Конфиг и логи `sing-box` сохраняются в `%APPDATA%/swanray/singbox/`:
- `config.json` — сгенерированный конфиг
- `singbox.log` — логи

## Известные ограничения

- Только Windows (split tunneling реализован под `process_name` Windows).  
  На Linux/macOS sing-box тоже поддерживает `process_name`, но потребуется адаптация запуска (`pkexec`/`sudo` вместо UAC).
- На каждое подключение/отключение Windows показывает UAC-промпт. Это можно убрать, поставив sing-box как Windows-службу (будущее улучшение).
- IPv6-маршрутизация настроена базово — может потребоваться тюнинг под ваш сервер.

## Сборка дистрибутива

Если в будущем нужно собрать `.exe`-установщик:

```bash
npm install -D electron-builder
npx electron-builder --win
```

Не забудьте включить папку `bin/` в `build.extraResources` в `package.json`.
